# 📌 Requisitos para Execução de Testes de Performance

*(Aplicável a Sistemas Web, APIs e Arquiteturas Distribuídas)*

---

## 1️⃣ Requisitos de Ambiente

O ambiente deve estar:

* Estável e sem deploys durante o teste
* Com infraestrutura documentada (CPU, RAM, disco, rede)
* Configurado o mais próximo possível da produção
* Com monitoramento habilitado
* Sem interferência de outros testes simultâneos

### Deve conter:

* Servidor Web (ex: Nginx, Apache, etc.)
* Servidores de aplicação
* Banco de dados
* Cache (se houver)
* Balanceador de carga
* Serviços externos mockados ou controlados

⚠ Testar em ambiente subdimensionado gera resultado inválido.

---

## 2️⃣ Requisitos de Dados

O banco deve conter:

* Volume realista de dados
* Índices configurados
* Estatísticas atualizadas
* Massa de teste variada

Problema comum:
Testar com banco vazio e concluir que sistema é rápido.

---

## 3️⃣ Requisitos de Infraestrutura

Monitorar obrigatoriamente:

* Uso de CPU
* Uso de memória
* I/O de disco
* Conexões abertas
* Pool de conexões de banco
* Threads ativas
* Latência de rede
* Saturação de containers (se houver)

Se for cloud:

* Autoscaling habilitado?
* Limites de instância configurados?
* Política de escalabilidade documentada?

---

## 4️⃣ Requisitos de Aplicação

A aplicação deve:

* Tratar erros adequadamente
* Implementar timeouts configuráveis
* Possuir logs estruturados
* Ter tratamento de exceções consistente
* Evitar chamadas síncronas bloqueantes desnecessárias

Importante verificar:

* Existência de cache
* Uso de fila (RabbitMQ, Kafka, etc.)
* Integrações externas (gateway de pagamento, APIs)

---

## 5️⃣ Requisitos de Arquitetura

Em sistemas modernos (microserviços), validar:

* Comunicação entre serviços
* Latência entre containers
* Balanceamento de carga
* Circuit breaker (se existir)
* Fallbacks

Performance não é só endpoint.
É fluxo completo.

---

## 6️⃣ Requisitos de Configuração do Servidor Web

Se houver servidor como Nginx:

* Limite de conexões simultâneas
* Timeout de requisição
* Compressão habilitada (gzip, brotli)
* Cache configurado
* Keep-alive ativo

Isso impacta diretamente o teste.

---

## 7️⃣ Requisitos de Segurança e Rate Limiting

Verificar:

* Limitação de requisições por IP
* Proteção contra flood
* Firewall de aplicação (WAF)
* Bloqueio automático sob carga excessiva

Às vezes o teste falha não por gargalo — mas por bloqueio de segurança.

---

## 8️⃣ Requisitos de Monitoramento e Observabilidade

Antes de rodar teste, deve estar ativo:

* Métricas de aplicação
* Logs centralizados
* Dashboards configurados
* APM (se disponível)
* Alertas desativados temporariamente (se necessário)

Sem observabilidade, não há diagnóstico.

---

## 9️⃣ Requisitos de Experiência do Usuário (Frontend)

Para páginas web:

* Tempo de carregamento total
* TTFB (Time To First Byte)
* Renderização completa
* Core Web Vitals
* Tempo de carregamento de assets
* CDN funcionando

Performance backend não garante boa experiência.

---

## 🔟 Requisitos de Definição de Métricas

Antes da execução, deve estar definido:

* Tempo máximo aceitável (p95 / p99)
* Throughput esperado
* Taxa máxima de erro
* Uso máximo aceitável de CPU/memória
* Tempo máximo de recuperação após falha

Sem meta → não existe conclusão.

---

# 📊 Pontos de Atenção (Nível Estratégico)

### 🔹 Tempo de Resposta

Avaliar média + percentis.

### 🔹 Manuseio de Erros e Timeouts

Sistema degrada ou quebra?

### 🔹 Resiliência ao Tráfego Sustentado

Mantém estabilidade por horas?

### 🔹 Limitação de Conexões

O servidor bloqueia antes de saturar?

### 🔹 Desempenho do Back-End

Banco é gargalo?
Query lenta?
Lock?

### 🔹 Desempenho com Cache

Com cache habilitado vs desabilitado.

### 🔹 Monitoramento e Logs

Existe correlação entre erro e uso de recurso?

### 🔹 Compatibilidade de Navegadores

Frontend reage diferente sob carga?

---

# 🧠 Mentalidade de QA de Performance

Antes de rodar qualquer teste, pergunte:

* O sistema tem limite configurado?
* Ele escala automaticamente?
* Se falhar, como falha?
* Existe plano de contingência?
* O negócio sabe o risco?

---

# 🚨 O que não pode faltar

✔ Ambiente documentado
✔ Métricas definidas
✔ Monitoramento ativo
✔ Volume realista de dados
✔ Escopo fechado
✔ Critério claro de sucesso
